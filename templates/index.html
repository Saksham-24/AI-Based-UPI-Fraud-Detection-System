How Scoring Works

Each transaction is passed as a 10-feature vector to the selected sklearn model. predict_proba(X)[0,1] returns the exact fraud probability — no approximation, no manual weights. The Random Forest aggregates 100 decision trees; each tree votes and the averaged probability becomes the fraud risk score.

Why Random Forest Wins

Random Forest aggregates 100 decision trees, each trained on a random feature/sample subset. The final predict_proba is the average probability across all trees — this averaging reduces variance and prevents the overfitting seen in a single Decision Tree.

Its high precision (~90%) means very few legitimate transactions are wrongly blocked, while an AUC of ~78% shows strong rank-order discrimination between fraud and genuine transactions across all classification thresholds.

10,000 row PaySim-style dataset generated with NumPy using realistic UPI transaction distributions (amount, time, frequency, flags).

Class imbalance corrected with upsampling (≈SMOTE). Categorical encoding, StandardScaler normalisation for LR, 80/20 stratified split.

Feature Engineering

10 behavioural features: spend deviation, frequency windows (1h/24h), night flag, device & merchant novelty signals, weekend flag.

Model Training

Four sklearn classifiers trained and compared. LR uses scaled features; tree-based models use raw features. All serialised with pickle.
